Choose Your Workflow

Three modes for different use casesâ€”from interactive development to batch processing.

Recommended

Persistent Mode

Keep the agent running for instant Claude access. Context persists across commands.

make up-agent PROJECT=~/game
make claude P="Add jump"
make claude P="Fix bug"
make down-agent
Async

Queue Mode

Drop task files in a directory and walk away. Perfect for overnight batch processing.

make queue-start PROJECT=~/game
echo "Add AI" > .claude/queue/001.md
make queue-status
make queue-stop
Isolated

One-Shot Mode

Fresh container each time. Maximum isolation for untrusted tasks or testing.

make run-direct PROJECT=~/game
# or for extra safety:
make run-staging STAGING=~/staging
make run-offline PROJECT=~/game

Quick Start

Get up and running in under 5 minutes.

1

Clone the Repository

git clone https://github.com/SapphireBeehiveStudios/godot-agent.git
cd godot-agent
2

Configure Authentication

Set up your Claude credentials (Claude Max or API key)

cp .env.example .env
# For Claude Max subscription:
claude setup-token
# Add token to .env: CLAUDE_CODE_OAUTH_TOKEN=sk-ant-...
3

Verify Environment

make doctor
make auth
4

Start Building

Launch the persistent agent with your Godot project

make build
make up-agent PROJECT=/path/to/your/godot/project
make claude P="What's in this project?"
